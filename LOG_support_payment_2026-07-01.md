## 6. Два сценария эксплуатации (как просили)

### Скрипт 1 — `deploy_stable.sh` (стабильно, со своего репо)
- Разворачивает бот, **зафиксированный** на проверенной версии (тег `stable-support-payment-v1`)
- Версия не «уедет» сама; не зависит от доступности/изменений upstream
- Делает бэкап (.env + дамп БД), health-check, при сбое — авто-откат
- Сменить версию: `PINNED_REF=<тег|коммит> bash deploy_stable.sh`

### Скрипт 2 — `update_safe.sh` (устойчиво к любым апдейтам)
- `--check` — посмотреть, есть ли апдейты upstream (без изменений)
- Без флага — влить upstream через **merge**:
  - **конфликт** → обновление отменяется, прод продолжает работать (ничего не ломается)
  - **чисто** → пересборка + health-check; при сбое — **авто-откат** на рабочую версию
- Бэкап БД/образа/.env перед каждым обновлением, уведомления в Telegram

> Стратегия «не сломается в будущем»: фича живёт в форке, апдейты автора вливаются
> git-merge'ем. Git сам сливает непересекающиеся изменения; при пересечении — безопасно
> останавливается, а не ломает прод. Фича вынесена в отдельные файлы (CRUD, миграция,
> модель в конце файла) → минимальная площадь конфликтов.

## 7. Продуманные риски и защита

| Риск | Решение |
|------|---------|
| Двойное зачисление (двойной клик «Подтвердить») | Проверка `status == 'pending'` перед обработкой |
| Гонки баланса | `add_user_balance` блокирует строку (`SELECT … FOR UPDATE`) |
| Подтверждение не-админом | `settings.is_admin(callback.from_user.id)` в обоих хендлерах |
| Поломка при обновлении бота | `update_safe.sh`: merge + health-check + авто-откат |
| Затирание фичи при `git pull` | Фича в форке; сервер тянет форк, а не upstream |
| Потеря данных при апдейте | Дамп БД + бэкап .env + сохранение образа перед каждым обновлением |
| Сломанная миграция | Идемпотентна (проверка существования таблицы), авто-применяется при старте |
| Секреты в логах | Токен читается из .env, в чат не выводится |

## 9. Известные ограничения / на будущее

  завершает покупку кнопкой «Завершить покупку» (1 тап). Полностью автоматическая

  используют дефолт-фолбэк). При желании — добавить переводы в locale-файлы.
- Поддержка форка: периодически запускать `update_safe.sh` для вливания апдейтов автора.
